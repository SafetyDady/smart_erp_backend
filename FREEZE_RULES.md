# Freeze Rules and Guardrails

## Overview

ในเฟสปัจจุบัน เราได้ทำการ freeze โครงสร้างบางส่วนของระบบเพื่อป้องกันไม่ให้เกิดการเปลี่ยนแปลงที่อาจส่งผลกระทบต่อสิ่งที่เสร็จสมบูรณ์แล้ว โดยมีเงื่อนไขสำคัญดังนี้:

## Freeze Conditions

1. **ห้ามนำ HTML pattern attribute กลับมาใช้ในฟอร์มที่เราได้แก้ไขไปแล้ว**

2. **ห้ามเปิดใช้งานฟังก์ชัน ADJUST ใน Stock Movement จนกว่าจะมีการอนุมัติเป็นทางการ**

3. **ห้ามให้ผู้ใช้กรอก Cost Center หรือ Cost Element ด้วยตนเองในฟอร์ม ISSUE หรือ Work Order (ต้องใช้ dropdown ที่กำหนดไว้เท่านั้น)**

4. **ห้ามแก้ไข schema หลักหรือเปลี่ยนแปลง logic ของ movement ที่เราได้กำหนดไว้แล้วในเฟสนี้**

## Purpose

เงื่อนไขเหล่านี้ถูกกำหนดขึ้นเพื่อรักษาความเสถียรของระบบ ป้องกันผลกระทบที่ไม่คาดคิดต่อส่วนที่ทำเสร็จแล้ว และเพื่อให้แน่ใจว่าการพัฒนาต่อไปจะอยู่ในขอบเขตที่กำหนดไว้
